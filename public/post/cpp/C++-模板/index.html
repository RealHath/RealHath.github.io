1 模板

1.1 基本语法

  1. 自动类型推导

template<typename T>    //声明一个模板，告诉编译器通用数据类型T
void mySwap(T &a, T &b)
{
    T temp = a;
    a = b;
    b = temp;
}
  1. 显示指定类型
1

mySwap<int>(a,b);

1.2 普通函数和函数模板的区别

  1. 普通函数调用可以发生隐式类型转换
  2. 函数模板用自动类型推导，不可以发生隐式类型转换
1

void mySwap(T &a, T &b)
  1. 函数模板用显示指定类型，可以发生隐式类型转换
1

mySwap<int>(a,b);

1.3 普通函数与函数模板的调用规则

  1. 如果函数模板和普通函数都可以调用，优先调用普通函数
1
2
3

int a, b;
void mySwap(int &a, int &b){}   //优先调用
void mySwap(T &a, T &b){}
  1. 可以通过空模板参数列表强制调用函数模板
1

mySwap<>(a,b);
  1. 函数模板可以发生函数重载（overload）
1
2

void mySwap(T &a, T &b){}
void mySwap(T &a, T &b, T &c){}
  1. 如果函数模板可以产生更好的匹配，优先调用函数模板
1
2
3

char a, b;
void mySwap(int &a, int &b){}   
void mySwap(T &a, T &b){}       //优先调用

  1. 类模板没有自动类型推导，只能用显示类型

2

三种传入方式：

  1. 指定传入类型（常用

1.8.1 类模板分文件编写

问题：类模板写在另一个cpp中，其他cpp要调用类模板中的方法，出现链接错误

原因：由于类模板中成员函数在程序运行后成员函数才被创建，而编译器要在链接是识别成员函数，相当于调用了一个没定义的函数，导致错误

解决方法：将头文件和函数写在同一个文件里，后缀名是.hpp，直观看出是类模板文件
